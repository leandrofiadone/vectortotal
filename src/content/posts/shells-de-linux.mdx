Al igual que Windows tiene la Símbolo del Sistema y PowerShell, Linux tiene distintos tipos de **shells** disponibles, cada una con sus propias características. La shell es el intérprete que procesa lo que escribís en la terminal y lo ejecuta.

Para ver cuál shell estás usando ahora mismo:

```bash
echo $SHELL
# /bin/bash

cat /etc/shells   # todas las instaladas en el sistema
```

### Cambiar de shell

```bash
# Temporalmente (solo en esta sesión)
zsh

# Permanentemente
chsh -s /usr/bin/zsh
```

<Callout type="info">
  El cambio permanente con `chsh` toma efecto recién al abrir una nueva sesión de terminal.
</Callout>

Bash surgió como reemplazo mejorado de `sh`, `ksh` y `csh`, tomando lo mejor de cada una. Sus características clave:

- **Tab completion** — presioná <kbd>Tab</kbd> para completar comandos, rutas y variables. Doble <kbd>Tab</kbd> muestra todas las opciones.
- **Historial** — flechas ↑ ↓ para navegar, `history` para ver todo, <kbd>Ctrl</kbd>+<kbd>R</kbd> para buscar de forma interactiva.
- **Scripting POSIX** — cualquier script `.sh` que encuentres en Internet va a correr en Bash sin modificaciones.

```bash
# Navegar historial de forma interactiva
Ctrl + R

# Ejecutar el último comando que usó nmap
!nmap

# Ver los últimos 20 comandos
history | tail -20
```

<Callout type="tip">
  En CTFs y pentesting, todos los reverse shells, one-liners y scripts de explotación asumen Bash. Es la más importante de las tres para aprender bien.
</Callout>

Fish prioriza la experiencia del usuario sobre todo lo demás:

- **Autosugerencias en tiempo real** — mientras escribís, Fish sugiere el comando más probable en gris. Presioná → para aceptar.
- **Resaltado de sintaxis built-in** — los comandos válidos aparecen en un color y los inválidos en otro. Detectás errores *antes* de ejecutar.
- **Autocorrección** — escribe `gti` y Fish te pregunta si quisiste decir `git`.
- **Personalización interactiva** — `fish_config` abre un panel web para cambiar temas, prompts y colores sin tocar archivos de configuración.

<Callout type="warning">
  Fish **no es compatible con scripts Bash/POSIX**. Si necesitás ejecutar un `.sh`, llamalo explícitamente: `bash script.sh`. Por eso no es ideal como shell principal en servidores.
</Callout>

Zsh combina la compatibilidad de Bash con características modernas:

- **Tab completion avanzado** — muestra un menú interactivo de opciones navegable con flechas, no solo completa a ciegas.
- **Globbing extendido** — patrones de búsqueda de archivos mucho más potentes que Bash.
- **oh-my-zsh** — el framework de plugins más popular: temas, aliases, integración con Git, completado de comandos para Docker, kubectl, etc.

```bash
# Instalar oh-my-zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# Plugins recomendados en ~/.zshrc
plugins=(git zsh-autosuggestions zsh-syntax-highlighting z)
```

<Callout type="tip">
  Si usás Kali Linux, Zsh ya viene instalado y preconfigurado. Es la shell que más uso — tiene la compatibilidad de Bash con la comodidad de Fish.
</Callout>

## 🎯 ¿Cuál usar?

- **En CTFs y pentesting** → **Bash**. Todos los exploits, reverse shells y scripts del ecosistema la asumen. Conocerla a fondo es no negociable.
- **En tu máquina de uso diario** → **Zsh** con oh-my-zsh. La compatibilidad de Bash más plugins útiles.
- **Empezando en Linux** → **Fish**. La curva de entrada más baja, el resaltado de sintaxis te salva de muchos errores.

<Callout type="danger">
  En servidores remotos o entornos que no controlás, **asumir que hay algo distinto a Bash puede romperte los scripts**. Siempre verificá con `echo $SHELL` antes de ejecutar.
</Callout>
